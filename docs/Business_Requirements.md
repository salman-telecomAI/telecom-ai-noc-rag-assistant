# Business Requirements

## Purpose

This document defines the business objectives and expected outcomes of the Telecom AI NOC RAG Assistant project.

The project aims to demonstrate how Artificial Intelligence can improve telecom operations by providing accurate, context-aware technical assistance using Retrieval-Augmented Generation (RAG).

---

# Business Problem

Telecom engineers spend significant time searching through technical documentation to investigate alarms and resolve network issues.

Manual searching can be:

- Time-consuming
- Inconsistent
- Error-prone

An AI assistant can reduce this effort by providing relevant information quickly.

---

# Business Objectives

The project should:

- Reduce troubleshooting time.
- Improve access to telecom knowledge.
- Demonstrate AI capabilities.
- Showcase modern AI architecture.
- Serve as a professional portfolio project.

---

# Expected Benefits

- Faster information retrieval
- Consistent technical responses
- Improved learning for telecom engineers
- Practical demonstration of AI technologies
- Better interview and portfolio presentation

---

# Stakeholders

Primary User

Telecom NOC Engineer

Secondary Users

- AI Engineers
- Solution Architects
- Recruiters
- Technical Interviewers

---

# Project Success Criteria

The project will be considered successful if it can:

- Answer telecom questions accurately.
- Retrieve relevant documentation using RAG.
- Generate structured AI responses.
- Run successfully using Docker.
- Be documented professionally on GitHub.

---

# Constraints

- Uses Azure OpenAI.
- Uses ChromaDB.
- Uses FastAPI.
- Designed as a portfolio project.
- No paid production infrastructure required.

---

# Key Takeaways

- Demonstrates practical AI engineering.
- Solves a real telecom knowledge retrieval problem.
- Provides measurable value for telecom operations.
- Serves as a professional portfolio project.
