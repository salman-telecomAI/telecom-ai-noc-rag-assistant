# Project Scope

## Purpose

This document defines the scope of the Telecom AI NOC RAG Assistant.

The scope identifies the features included in the current version of the project and clarifies what is intentionally excluded.

The objective is to keep the project focused, maintainable and suitable as a professional AI portfolio.

---

# Project Goal

Develop an AI-powered Telecom Network Operations Center (NOC) Assistant that uses Retrieval-Augmented Generation (RAG) to answer telecom engineering questions using a custom knowledge base.

---

# In Scope

The current project includes:

## Artificial Intelligence

- Azure OpenAI integration
- Retrieval-Augmented Generation (RAG)
- Structured AI responses
- Telecom-specific knowledge retrieval

# Out of Scope

The following features are intentionally excluded from Version 1.0:

- User authentication
- User authorization
- Web frontend
- Mobile application
- Database authentication
- Chat history
- Multi-user support
- CI/CD pipeline
- Kubernetes
- Terraform
- Monitoring dashboards
- Production cloud deployment
- High availability
- Load balancing

These features may be considered for future versions if required.

# Success Criteria

The project is considered successful if it:

- Answers telecom questions accurately.
- Retrieves relevant knowledge using RAG.
- Generates structured AI responses.
- Runs successfully using FastAPI.
- Runs successfully using Docker.
- Is professionally documented on GitHub.

---

# Assumptions

The project assumes:

- Azure OpenAI services are available.
- Telecom documents have been indexed.
- Required environment variables are configured.
- Docker Desktop is installed for container deployment.
